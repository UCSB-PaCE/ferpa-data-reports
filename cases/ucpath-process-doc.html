UCSB use case

Process job aids

For Operations and training leads

You have

Several separate job aids for the same workflow, each written differently

↓
You want

One consolidated process document with gaps and recommended improvements

How it works

The same pattern the whole guide teaches, applied to a stack of overlapping job aids: a script gathers and numbers the source documents, the model is brought in only to merge them into one ordered process and name the gaps, and a person confirms before it becomes training material.

  1. 1
    You

    Gather the source job aids

    Collect every job aid that covers this workflow into one folder, one file per aid (the documents themselves are the free text). This is the only step that touches the raw source material, and it stays on your machine.

  2. 2
    Script

    Let code collect and number them

    A script reads each job aid, gives every aid a stable source_id and every step or instruction a numbered line_id, and stacks them into one table. The collecting, labeling, and ordering stay in code, where they are exact and repeatable, never in the model.

    Prompt to paste 
    Write a Colab cell that uses pandas to load every job aid in a folder (start with {your job-aids folder, e.g. job_aids/}, reading .txt/.md/.docx) into a DataFrame called df, one row per step or instruction line. Give each source file an integer source_id and each row an integer line_id so every instruction can be cited later, and keep a column for the source filename. Drop fully empty lines. Print df.head() and a count of lines per source_id. Do all the collecting and numbering in pandas, never in a model.
  3. 3
    AI, the one hard part

    Merge them into one ordered process

    The model reads the numbered instructions from all the aids and writes a single, ordered process: it reconciles the different wordings, names where the aids conflict or leave gaps, proposes improvements, and lists clarifying questions for a human. Numbered instructions in, one cited draft process out.

    Prompt to paste 
    Write a Colab cell that sends only the source_id, line_id, and instruction text (never any names or login credentials) to the LLM Sandbox gateway (an OpenAI-compatible endpoint; read the base URL and API key from Colab Secrets, never hardcode them). Ask it to merge the overlapping aids into one ordered process with numbered steps, citing the source line_ids each step came from in square brackets like [3]; then, under separate headings, to list {the gaps, e.g. steps no aid covers}, where the aids conflict, {recommended improvements}, and a short list of clarifying questions for a human. Tell it to reply "sources disagree" rather than guess when two aids contradict each other. Collect the result as text. Never paste raw aids into this chat.
  4. 4
    A person

    Confirm before it becomes training

    An operations or training lead checks the merged process against its cited source lines, answers the model's clarifying questions, resolves the flagged conflicts, and decides which improvements to keep. The human stays at the gate; the draft is not training material until a person signs off.

  5. 5
    Output

    Out comes the consolidated process

    One ordered process document with each step traceable back to its source aids, a named list of gaps and conflicts, and recommended improvements, ready for a person to finalize as the single authoritative job aid.

    Prompt to paste 
    Write a Colab cell that writes the merged process to one document with the ordered steps first, then the gaps, conflicts, and recommended-improvements sections, rendering each [line_id] as a footnote that shows the verbatim source instruction and its filename. Save it to a process_doc/ folder I can download and print the path. Make the format {Markdown or .docx}.
