Any organization use case

Grant and program outcomes

For Program managers and grant reporters

You have

A roster of funded activities with metrics and free-text progress updates

↓
You want

A funder-ready outcomes report tying activities to results

How it works

The same pattern the whole guide teaches, applied to grant reporting: scripts roll up the metrics by goal, the model is brought in only to turn the free-text progress updates into a cited narrative against targets, and a program manager edits before it goes to the funder.

  1. 1
    You

    Get the activity roster out

    Export your funded activities to a spreadsheet: one row per activity, the metric columns (goal, target, actual), and the free-text progress_note column. This is the only step that touches the raw, possibly protected, data.

  2. 2
    Script

    Let code roll up the metrics

    A script totals the targets and actuals, computes attainment against each target, and groups everything by goal. The funder's numbers stay in code, where they are exact and repeatable, never in the model.

    Prompt to paste 
    Write a Colab cell that uses pandas to load my CSV (start with grant_outcomes_sample.csv) into a DataFrame called df. Clean it: lowercase and snake_case the column names, drop fully empty rows, and keep the activity_id on every row so each progress note can be cited later. Compute a percent_of_target column as actual divided by target times 100, rounded to one decimal. Then build a summary table grouped by {the unit you roll up to, e.g. goal} with the sum of target, the sum of actual, and the overall percent of target attained, rounded to one decimal. Print df.head() and the summary table. Do all the math in pandas, never in a model.
  3. 3
    AI, the one hard part

    Turn the progress notes into a cited narrative

    The model reads the free-text progress updates and writes a balanced outcomes narrative against the targets, where every claim links back to the activity it came from. Numbered notes in, cited narrative out. It never sees who wrote what.

    Prompt to paste 
    Write a Colab cell that, for each {goal, e.g. Retention}, gathers that goal's activities and sends only the activity_id, the percent_of_target number, and the progress_note text to the LLM Sandbox gateway (an OpenAI-compatible endpoint; read the base URL and API key from Colab Secrets, never hardcode them). Ask for a fair, {N, e.g. 4}-sentence outcomes narrative that ties activities to results against their targets, cites the activity_ids it used in square brackets like [ACT-016], names no individuals, and replies "not enough activities" when a goal has fewer than {min, e.g. 2} activities. Collect the results into a dict keyed by goal. Never paste raw progress notes into this chat.
  4. 4
    A person

    Edit it before submission

    The program manager checks the narrative against its cited activities and the rolled-up numbers, edits the framing and tone for the funder, and only then submits. The human stays at the gate; the model never has the last word.

  5. 5
    Output

    Out comes the funder-ready report

    A single outcomes report: the metrics rolled up by goal, each goal's cited narrative of what was achieved against target, ready to drop into the funder's reporting template.

    Prompt to paste 
    Write a Colab cell that builds one outcomes report with the summary table by goal first and each goal's cited narrative below it, rendering every [activity_id] as a footnote that shows the verbatim progress_note and its percent of target. Save the report as a file in a reports/ folder I can download, and print the path written. Make the format {Markdown or .docx}.
