UCSB use case

Surveys and evaluations

For program teams and faculty committees

You have

Survey and evaluation exports with ratings plus free-text comments (Qualtrics, REDCap, and the like)

↓
You want

Cited summaries: many per-instructor and per-department reports, or one executive summary with prioritized themes

How it works

The same pattern the whole guide teaches, applied to survey data: scripts own the numbers, the model is brought in only to read the comments, and a person approves before anything ships.

  1. 1
    You

    Get the responses out

    Export the survey or evaluation responses to a spreadsheet: one row per response, the rating columns, and the free-text comment columns. This is the only step that touches the raw, possibly protected, data.

  2. 2
    Script

    Let code do the counting

    A script computes every average, response rate, and breakdown (per instructor, per course, per segment) and absorbs the export's shifting shape. Math stays in code, where it is exact and repeatable, never in the model.

    Prompt to paste (no-code path) 
    Write a Colab cell that reads course_evals_sample.csv, standardizes the column names, drops empty rows, and numbers each comment so it can be cited later. In the same cell, build a per-instructor table of the average overall_rating and the percent who would recommend. Show the first five rows and the table.
  3. 3
    AI, the one hard part

    Summarize the comments, with citations

    The model reads the numbered free-text comments and writes a fair, balanced summary where every claim links back to the comment it came from. Numbered comments in, cited summary out. It never sees who wrote what.

    Prompt to paste (no-code path) 
    Write a cell that sends each instructor's numbered comments to the LLM Sandbox gateway (not the Colab helper) and returns a fair, three-sentence summary that cites the comment numbers it used, like [2] or [5], with no names. Keep the raw comments out of this chat.
  4. 4
    A person

    Read it and approve

    You spot-check the summary against its cited comments, fix anything off, and only then release it. The human stays at the gate; the model never has the last word.

  5. 5
    Output

    Out comes the report

    A stack of small reports (one per instructor) or a single executive summary with prioritized themes, ready to send to the people who need it.

    Prompt to paste (no-code path) 
    Write a cell that assembles one report per instructor (the score table first, then the cited comment summary) and saves each as its own file I can download.
