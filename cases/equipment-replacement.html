UCSB use case

Equipment replacement cycle

For IT asset and budget owners

You have

Asset inventory exported from a service desk as a CSV

↓
You want

A clean list of what needs replacing this fiscal year, with a short budget summary

How it works

The same pattern, applied to an IT asset inventory: scripts decide what is due for replacement and add up the cost, the model only writes the budget-cycle brief that frames the list, and a person approves before it goes to finance.

  1. 1
    You

    Export the inventory

    Pull the asset inventory out of your service desk as a CSV: one row per asset, with the fields that decide a replacement (purchase or in-service date, model, category, location or owner, and cost or replacement cost). This is the only step that touches the raw inventory.

  2. 2
    Script

    Let code flag what is due and add it up

    A script computes each asset's age, marks the ones whose replacement cycle lands in this fiscal year, and totals the cost by category and overall. The filtering and the math stay in code, where they are exact and repeatable, never in the model.

    Prompt to paste 
    Write a Colab cell that uses pandas to load my asset inventory CSV (start with asset_inventory_sample.csv) into a DataFrame called df. Clean it: lowercase and snake_case the column names, parse the {date the clock starts from, e.g. in_service_date} as dates, and add an integer asset_id to every row. Compute each asset's age in years, then flag rows due for replacement this fiscal year using age >= {replacement cycle in years, e.g. 4} (let me also pass an explicit {fiscal year window, e.g. 2026-07-01 to 2027-06-30}). Build a due_list table sorted by category, and a totals table with the count and summed {cost column, e.g. replacement_cost} per category plus a grand total, rounded to whole dollars. Print df.head(), the due_list, and the totals. Do all the math in pandas, never in a model.
  3. 3
    AI, the one hard part

    Write the budget-cycle brief over the numbers

    There is no free text here, so the model is brought in for the one thing a script does not do well: turning the totals into a short, plain-language budget narrative a fiscal owner can read. It writes only over the computed figures (counts and category totals), not over any raw inventory row.

    Prompt to paste 
    Write a Colab cell that sends only the computed totals table (category, count, and summed cost, plus the grand total) to the LLM Sandbox gateway (an OpenAI-compatible endpoint; read the base URL and API key from Colab Secrets, never hardcode them). Ask for a {N, e.g. 4}-sentence budget-cycle brief that states the total replacement cost for the fiscal year, calls out the {top K, e.g. 2} categories driving it, and frames it as an end-of-year request. Tell it to use only the numbers provided, invent no figures, and name no individual assets or people. Save the returned text as the brief. Never paste raw inventory rows into this chat.
  4. 4
    A person

    Check the numbers and approve

    You confirm the totals in the brief match the script's tables, adjust the framing if needed, and only then send it on. The human stays at the gate; the model never has the last word on a budget request.

  5. 5
    Output

    Out comes the replacement plan

    A clean, sorted list of what needs replacing this fiscal year, the cost totals by category, and a short budget summary on top, ready to hand to finance for the end-of-year request.

    Prompt to paste 
    Write a Colab cell that builds one report with the budget-cycle brief first, the totals table next, and the full due_list table below. Save it as a file in a reports/ folder I can download, and print the path. Make the format {Markdown, .docx, or .xlsx with the list and totals on separate sheets}.
