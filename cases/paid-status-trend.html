UCSB use case

Paid-status trends

For Registration and fee-status teams

You have

Daily registered-and-paid counts across many years of fall terms (summary only)

↓
You want

Census-aligned trend charts and a read on whether students are paying slower

How it works

The same pattern the whole guide teaches, applied to fee-status counts: scripts pivot every daily count to the census date and draw the trend lines, the model is brought in only to write the short readout of what the curves mean, and a person approves before it reaches leadership.

  1. 1
    You

    Get the daily counts out

    Export the daily registered-and-paid counts to a spreadsheet: one row per day, with the term (for example fall 2021), the calendar date, and the count of registered-and-paid students. Summary numbers only, no student rows. This is the only step that touches the source data, and because it is counts and not individual records, there is nothing identifying to send anywhere downstream.

  2. 2
    Script

    Let code align every year to census

    A script reindexes each term's daily counts to days-relative-to-census (census = day 0), so every fall lines up on the same axis no matter the calendar date. It then computes, for each day offset, the minimum, maximum, and current-year value across all terms, and draws the trend chart with those three lines. The pivoting and the math stay in code, where they are exact and repeatable, never in the model.

    Prompt to paste 
    Write a Colab cell that uses pandas to load my daily counts CSV (columns like term, date, paid_count) into a DataFrame called df. For each {term} compute days_to_census = (date - {census_date for that term, e.g. the official fall census}).dt.days so census is day 0, then pivot to a table indexed by days_to_census with one column per term. For each day offset compute the min, max, and the {current term, e.g. fall 2025} value across terms, and plot all three as lines with day 0 marked. Print the pivot table and save the chart to a file. Do all the alignment and math in pandas, never in a model.
  3. 3
    AI, the one hard part

    Write the readout of the curve

    The model takes the numbers the script computed (the current-year line against the historical min, max, and median, and how far before census each crosses key thresholds) and writes a short, plain-language brief: is this year pacing ahead of, behind, or in line with prior falls, and is the gap widening as census nears. Computed numbers in, a tight narrative out. It never sees a student record, only the aggregate figures.

    Prompt to paste 
    Write a Colab cell that sends only the computed summary numbers (current-term paid_count by days_to_census, plus the historical min, max, and median at the same offsets, and the percent of the eventual census total reached at, say, 14 and 7 days out) to the LLM Sandbox gateway (an OpenAI-compatible endpoint; read the base URL and API key from Colab Secrets, never hardcode them). Ask for a {N, e.g. 4}-sentence readout that states whether {current term, e.g. fall 2025} is pacing ahead of, behind, or in line with prior falls, whether students appear to be paying slower (the curve shifting later toward census), and one watch-item for leadership. Pass numbers only, no student rows, and never paste raw records into this chat.
  4. 4
    A person

    Check it against the chart

    You read the brief next to the chart, confirm it matches what the lines actually show, fix any over-reading of normal year-to-year wobble, and only then release it. The human stays at the gate; the model never has the last word on what the trend means.

  5. 5
    Output

    Out comes the trend brief

    A census-aligned chart with the min, max, and current-year lines, paired with a short readout of whether students are paying slower, ready to drop into a leadership update or a fee-status review.

    Prompt to paste 
    Write a Colab cell that builds one report with the census-aligned chart at the top (the min, max, and {current term} lines, day 0 marked) and the approved readout below it, followed by a small table of paid_count by days_to_census for the current term versus the historical median. Save it as a single file in a reports/ folder I can download, and print the path. Make the format {Markdown or .docx}.
