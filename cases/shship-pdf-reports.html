UCSB use case

Static PDF reports

For Analytics and finance leads

You have

Dense multi-page PDF reports from a campus partner, where only a few pages matter

↓
You want

Charts, trends, and the few nuggets of insight that drive a decision

How it works

The same pattern, applied to dense PDF reports: a script pulls out the tables that matter, the model writes the short trend narrative over those numbers, and an analyst verifies it against the source pages before it goes up.

  1. 1
    You

    Gather the source PDFs

    Drop the multi-page PDF reports from your campus partner into one folder, and note which pages or table titles actually carry the few numbers that drive the decision. This is the only step that touches the raw, possibly protected, source documents.

  2. 2
    Script

    Let code extract the tables

    A script opens each PDF, pulls out the relevant tables into a tidy spreadsheet, and computes the period-over-period changes and totals. Extraction and math stay in code, where they are exact and repeatable, and every figure keeps a citation back to the page it came from.

    Prompt to paste 
    Write a Colab cell that uses pdfplumber (with pandas) to open each PDF in a pdfs/ folder and extract only the tables on {the pages that matter, e.g. pages 3 and 7} or matching {a table title, e.g. "Quarterly enrollment by program"}. Concatenate them into one DataFrame called df, normalize the column names to snake_case, coerce the numeric columns to numbers, and add a source_id column recording the file name and page number for each row so every figure can be cited later. Then compute the period-over-period change and percent change for each metric, rounded to one decimal, into a summary table. Print df.head() and the summary table. Do all extraction and math in code, never in a model.
  3. 3
    AI, the one hard part

    Write the trend narrative over the numbers

    The model takes the already-computed numbers and writes a short, plain-language brief: the trend, the few nuggets that matter, and a flag on any standout that breaks the pattern. Numbers in, narrative out. It never sees the raw PDFs.

    Prompt to paste 
    Write a Colab cell that turns the summary table into a compact text block (metric name, current value, prior value, percent change, and its source_id) and sends only that text to the LLM Sandbox gateway (an OpenAI-compatible endpoint; read the base URL and API key from Colab Secrets, never hardcode them). Ask for a {N, e.g. 4}-sentence trend brief that names the {top, e.g. 2} standouts (the largest moves or anything crossing {a threshold, e.g. plus or minus 10 percent}), cites the source_id behind each claim in square brackets like [report.pdf p7], invents no figures, and uses only the numbers provided. Store the brief in a variable. Never paste raw PDF pages into this chat.
  4. 4
    A person

    Verify against the source pages

    An analyst spot-checks each figure and flagged standout against the cited page in the original PDF, corrects anything off, and only then signs off. The human stays at the gate; the model never has the last word on what the numbers mean.

  5. 5
    Output

    Out comes the brief

    A short decision-ready report: a clean chart of the key trends, the extracted numbers, and the cited narrative that points straight to the handful of pages that mattered, with standouts flagged.

    Prompt to paste 
    Write a Colab cell that builds one report combining a chart of the key metrics over the periods (matplotlib), the summary table, and the trend brief, rendering each [source_id] as a footnote so a reader can jump to the original page. Save it as {a file, e.g. trend_brief.docx or .md} in a reports/ folder I can download, and print the path written.
