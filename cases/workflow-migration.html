UCSB use case

Right tool for the workflow

For Form and workflow administrators

Used AI to translate complex signature-tool forms into automated process maps and workflows in a better-fit tool, with a human finalizing, eliminating recurring routing issues.

Nintex AIGemini

How it works

A signature tool kept misrouting approvals, so AI mapped the tangled forms into clean process maps and rebuilt them as workflows in a better-fit tool, with a person finalizing every map before it went live.

  1. 1
    You

    Describe the form and its routing

    Write down what the signature-tool form actually does: the fields it collects, who approves in what order, the branches ("if amount over X, add a second approver"), and the routing problems that keep recurring. Export the form's structure or a process diagram if the old tool can produce one. This is plain process description, no protected records, just how approvals are supposed to flow.

  2. 2
    AI

    Translate the form into a clean process map

    Hand the model your description and ask it to produce a structured process map: the steps in order, every decision branch with its condition, each approver role (by role, not by person), and the data each step needs. The model does the one thing that is genuinely hard here, untangling implicit, inconsistent routing logic into an explicit map you can rebuild from. It works only from your description of the process, never from real submissions or approver records.

    Prompt to paste 
    Set up the assistant (Nintex AI / Gemini) with this instruction, then paste your process description as the input. System instruction: "You are a process-mapping analyst. From a plain-language description of an approval form, produce a structured process map for rebuilding it in a workflow tool. Output, in order: (1) the linear steps; (2) every decision branch as an explicit condition (if {field} {operator} {value} then {route}); (3) each approver as a ROLE, never a named individual; (4) the fields each step reads or writes; (5) a list of routing ambiguities or gaps you found and the assumption you made for each. Ask a clarifying question before guessing when a branch is unclear. Use no real names, IDs, or record contents; work only from the process logic I give you."
  3. 3
    You

    Rebuild it in the better-fit tool, then finalize

    Use the map to rebuild the workflow in the tool that actually fits it (here, Nintex), wiring each step, branch, and approver role. Then a person finalizes: walk the map against real cases, confirm every branch routes correctly, fix the ambiguities the model flagged, and only then publish. The model proposed the structure; you own what goes live, which is exactly where the recurring routing issues get closed out.

Build it, your way

This one leans developer and tool-setup, not the Colab data pipeline: build it inside your workflow tool (Nintex), using AI to draft the process map and a person to finalize each branch before publishing.
