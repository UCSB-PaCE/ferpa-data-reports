UCSB use case

Admissions review

For Admissions committees and program coordinators

You have

Hundreds of applications with letters of intent and faculty review notes

↓
You want

A criteria-based scoring rubric, overlooked-applicant flags, and an admit-success readout

How it works

The same pattern the whole guide teaches, applied to admissions: scripts own the criteria-based scores, the model is brought in only to read the letters of intent and review notes, and a person approves before any applicant is flagged or admitted.

  1. 1
    You

    Get the applications out

    Export the applicant pool to a spreadsheet: one row per applicant, an applicant_id, the structured fields you score on (GPA, a preparation index, a resources-context flag), an outcome_marker for applicants you can already track, and the free-text columns (letter of intent, faculty review notes). This is the only step that touches the raw, possibly protected, data.

  2. 2
    Script

    Let code apply the rubric

    A script applies your scoring rubric to the structured fields the exact same way for every applicant, computes a composite score, and absorbs the export's shifting shape. The criteria and the math stay in code, where they are explicit, repeatable, and auditable, never in the model.

    Prompt to paste 
    Write a Colab cell that uses pandas to load my CSV (start with admissions_review_sample.csv) into a DataFrame called df. Clean it: lowercase and snake_case the column names, drop fully empty rows, and confirm applicant_id is unique so each application can be cited later. Then apply a transparent rubric in pandas: normalize each scoring field ({the fields you weight, e.g. gpa, prep_index}) to a 0-100 scale, combine them into a composite_score with weights I can edit at the top of the cell as a dict like {weights, e.g. {'gpa':0.5,'prep_index':0.5}}, and print the rubric weights plus df sorted by composite_score. Do all the scoring math in pandas, never in a model.
  3. 3
    AI, the one hard part

    Read the letters for overlooked strengths, with citations

    The model reads each applicant's letter of intent and faculty review notes and surfaces the qualities a numeric rubric misses (resilience, initiative, context behind the numbers), with every observation tied back to the line it came from. Text in, cited flags out. It never sees applicant names, only the ids.

    Prompt to paste 
    Write a Colab cell that, for each applicant_id, sends only the applicant_id and that applicant's letter_of_intent and review_notes text to the LLM Sandbox gateway (an OpenAI-compatible endpoint; read the base URL and API key from Colab Secrets, never hardcode them). Ask for a fair, {N, e.g. 3}-sentence read of non-numeric strengths and a single overlooked flag (true or false) for applicants whose letters show strong qualities their composite_score does not capture, citing the applicant_id in square brackets like [A-2271], naming no individuals, and replying "not enough text" when an applicant has fewer than {min, e.g. 2} sentences of free text. Collect the results into a dict keyed by applicant_id. Never paste raw applications into this chat.
  4. 4
    A person

    Review the flags and approve

    The committee spot-checks each rubric score and each overlooked-applicant flag against its cited text, corrects anything off, and only then acts on it. The human stays at the gate; the model never decides an admission and never has the last word.

  5. 5
    Output

    Out comes the readout

    A ranked rubric sheet, a short list of overlooked applicants the numbers alone would have passed over, and an admit-success readout that holds rubric scores against the outcome_marker so you can see which criteria actually predict who thrives.

    Prompt to paste 
    Write a Colab cell that builds the admissions readout: a ranked table by composite_score with the overlooked flag and its cited note shown inline, a separate short list of applicants flagged overlooked, and a calibration view that groups applicants by composite_score band and reports the share with each outcome_marker (e.g. thriving) so we can see how well the rubric predicts success. Save it as {Markdown or .docx} in a reports/ folder I can download, and print the list of files written.
