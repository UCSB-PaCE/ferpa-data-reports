Higher ed use case

Advising notes

For Advising teams and student success offices

You have

A term of advisor session notes with satisfaction scores and response times

↓
You want

A per-advisor summary of strengths, themes, and follow-up gaps

How it works

The same pattern, applied to a term of advising notes: scripts compute every average from the scores, the model is brought in only to read the free-text notes and write a fair, cited per-advisor summary, and a person approves before it ships.

  1. 1
    You

    Get the notes out

    Export the term's advising sessions to a spreadsheet: one row per session, the advisor column, the satisfaction score, the response-time and session-length columns, and the free-text "Notes / summary" column. This is the only step that touches the raw, possibly protected, data, including advisor names.

  2. 2
    Script

    Let code do the counting

    A script computes each advisor's averages (mean satisfaction, mean response time in hours, mean session length in minutes) and their session count, and it absorbs the export's shifting shape. Math stays in code, where it is exact and repeatable, never in the model.

    Prompt to paste 
    Write a Colab cell that uses pandas to load my CSV (start with advising_notes_sample.csv) into a DataFrame called df. Clean it: lowercase and snake_case the column names (so "Satisfaction (1-5)" becomes something like satisfaction_1_5 and "Response time (hrs)" becomes response_time_hrs), drop fully empty rows, and add an integer note_id to every row so each free-text note can be cited later. Then build a summary table grouped by {the unit you want one report per, e.g. advisor} with the session count and the mean of satisfaction, response_time_hrs, and session_length_min, rounded to one decimal. Print df.head() and the summary table. Do all the math in pandas, never in a model.
  3. 3
    AI, the one hard part

    Summarize the notes, with citations

    The model reads each advisor's numbered free-text notes and writes a fair, balanced summary of what students valued, the recurring themes, and where follow-up slipped, with every claim linked back to the note it came from. Numbered notes in, cited summary out. It never sees who wrote what.

    Prompt to paste 
    Write a Colab cell that, for each {group, e.g. advisor}, gathers that group's numbered notes and sends only the note_id and the note text (the "Notes / summary" column) to the LLM Sandbox gateway (an OpenAI-compatible endpoint; read the base URL and API key from Colab Secrets, never hardcode them). Ask for a fair, {N, e.g. 3}-sentence summary covering strengths, recurring themes, and any follow-up gaps, that cites the note_ids it used in square brackets like [12], names no individuals, and replies "not enough notes" when a group has fewer than {min, e.g. 3} notes. Collect the results into a dict keyed by group. Never paste raw notes or advisor names into this chat.
  4. 4
    A person

    Read it and approve

    You spot-check each summary against its cited notes and the computed averages, fix anything off, and only then release it. The human stays at the gate; the model never has the last word.

  5. 5
    Output

    Out comes the report

    A stack of small per-advisor summaries, each pairing the averages (satisfaction, response time, session length) with the cited narrative of strengths, themes, and follow-up gaps, ready to share for coaching and review.

    Prompt to paste 
    Write a Colab cell that builds one report per {group, e.g. advisor} with the averages table first (satisfaction, response_time_hrs, session_length_min, session count) and the cited note summary below, rendering each [note_id] as a footnote that shows the verbatim note. Save each report as its own file in a reports/ folder I can download, and print the list of files written. Make the format {Markdown or .docx}.
