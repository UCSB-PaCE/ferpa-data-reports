UCSB use case

Advising training, simulated

For Lean advising teams training peer advisors

Turned real, redacted advising scenarios into an interactive training module with AI-avatar video, so new advisors can practice safely instead of learning on live students.

LLM SandboxGoogle Vids

How it works

The same pattern, applied to advising practice: scripts and people own the real scenarios, the model is brought in only to script and play the student, and a person approves every scenario before a peer advisor ever runs it.

  1. 1
    You

    Redact the real scenarios

    Start from real advising situations your team has actually handled, then strip them down to the lesson: the student's question, the constraints, and the right and wrong moves. Remove every name, ID, and identifying detail so what is left is a generic, teachable scenario. This is the only step that touches real, possibly protected, history, and it stays in your hands, never in the model.

  2. 2
    AI

    Have AI write the practice dialogue

    Feed the redacted scenarios to the model and have it write each one up as a branching practice script: an opening from the student, a few realistic ways the advisor might respond, and where each path leads. The model plays the student and drafts the coaching notes; it only ever sees the scrubbed scenarios, never the real records they came from.

    Prompt to paste 
    Write a system prompt and a per-scenario user prompt for the LLM Sandbox gateway (an OpenAI-compatible endpoint; read the base URL and API key from Colab Secrets or your tool's secret store, never hardcode them) that turns one redacted advising scenario into a branching practice script. The system prompt should cast the model as a {student persona, e.g. anxious first-quarter transfer}, hold it to {your advising policy, e.g. FERPA limits and referral rules}, and forbid inventing facts not in the scenario. The user prompt sends only the scrubbed scenario text and asks for: a 2 to 3 line student opening, {N, e.g. 3} advisor response options labeled good / risky / wrong with a one-line consequence each, and short coach notes citing the scenario point each option maps to. Return it as structured JSON so a person can review it. Never paste real student records into this chat.
  3. 3
    A person

    Approve each scenario

    An experienced advisor reads every generated scenario against your real policy: is the right answer actually right, are the referrals correct, is anything misleading. They edit or cut what is off, and only approved scenarios move forward. The human stays at the gate; the model never decides what good advising looks like.

  4. 4
    Output

    Record it as an interactive module

    Take the approved scripts into Google Vids and build the module: an AI-avatar plays the student, the practice paths become the interactive choices, and your coach notes become the feedback. New peer advisors practice the hard conversations safely, on simulated students, before they ever sit with a real one.
