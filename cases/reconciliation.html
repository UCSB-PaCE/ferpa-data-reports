UCSB use case

Transaction reconciliation

For Department financial analysts

You have

Hundreds of multi-line card transactions (charges, reversals, allocations) across several reports

↓
You want

An exception report flagging what needs review, and optionally the whole match automated

How it works

The same pattern the whole guide teaches, applied to card reconciliation: scripts do every match and every sum, the model is brought in only to write the plain-language note for each flagged mismatch, and a person approves before anything ships.

  1. 1
    You

    Get the transactions out

    Export the card transactions to a spreadsheet: one row per line, with the transaction id, posting date, vendor, amount, and allocation status. Pull every report you reconcile across (charges, reversals, allocations) into the same shape. This is the only step that touches the raw, possibly protected, financial data.

  2. 2
    Script

    Let code do the matching and the math

    A script groups the charge, reversal, and allocation lines, sums each group, and flags every line where the numbers do not net to zero or the allocation is short. All matching and arithmetic stay in code, where they are exact and repeatable, never in the model. Out comes a clean list of exceptions with the amounts already computed.

    Prompt to paste 
    Write a Colab cell that uses pandas to load my CSV (start with flex_card_reconciliation_sample.csv) into a DataFrame called df. Clean it: lowercase and snake_case the column names, parse amount to a float and posting_date to a date, and add an integer txn_ref to every row so each flagged line can be cited later. Group the related lines by {the key that ties a charge to its reversal and allocation, e.g. vendor + posting_date}, sum the amount per group, and flag a group as an exception when the net is not zero (beyond a {tolerance, e.g. 0.01} cent rounding tolerance) or the allocation_status is not "allocated". Build an exceptions table with the txn_ref, the group key, the computed net, and a short reason code. Print df.head() and the exceptions table. Do all the matching and math in pandas, never in a model.
  3. 3
    AI, the one hard part

    Write the plain-language note for each flag

    The model takes the numbers the script already computed for each flagged group (the net, the reason code, the line count) and writes one clear sentence an analyst can read at a glance, like "reversal recorded with no matching original charge". It does the one thing a script cannot: turn a reason code and a few figures into readable English. It never does the math and never sees who spent what.

    Prompt to paste 
    Write a Colab cell that, for each flagged group, sends only the computed figures (the txn_ref, the reason code, the net amount, and the line count) to the LLM Sandbox gateway (an OpenAI-compatible endpoint; read the base URL and API key from Colab Secrets, never hardcode them). Ask for one plain-language sentence, at most {N, e.g. 25} words, that explains what an analyst should review, references the txn_ref in square brackets like [14], names no vendor or person, and states no dollar figure the script did not provide. Collect the notes into a dict keyed by txn_ref. Never paste raw transaction rows into this chat; send only the numbers the script computed.
  4. 4
    A person

    Review the exceptions and approve

    An analyst reads each flagged line against its note, confirms the real mismatches, clears the false alarms, and only then signs off. The human stays at the gate; the model never decides what counts as a problem.

  5. 5
    Output

    Out comes the exception report

    A short, sorted exception report: only the lines that need review, each with the computed net and a one-sentence note, ready to hand to the analyst or feed into the next reconciliation pass.

    Prompt to paste 
    Write a Colab cell that builds an exception report from the exceptions table and the notes dict, sorted by {priority, e.g. largest net first}. For each flagged line, show the txn_ref, the group key, the computed net, the reason code, and the plain-language note. Save it as a file I can download (make the format {Markdown or .xlsx}) and print the count of lines flagged. Include only the flagged lines, not the full ledger.
